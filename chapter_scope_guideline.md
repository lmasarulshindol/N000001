# プロローグおよび第一章の描写範囲 定義ガイド

このガイドは、物語冒頭でありがちな「情報過多」「フック不足」「人物理解不足」を避けるために、
**プロローグ**と**第一章**で「どこまで描くか」を明確にするための基準です。

---

## 1. プロローグで描く範囲（上限）

### 目的
- 読者に「この物語は何が起きそうか」を直感させる。
- 物語全体の空気（緊張・不穏・期待）を先に提示する。

### 描いてよい要素
- [ ] 異常事態・事件・象徴シーンを**1つ**。
- [ ] 世界観情報は「理解に必要な最低限」だけ。
- [ ] 本編で回収される問い（謎）を**1〜2個**。
- [ ] 語り手が本編主人公でない場合、その理由が後で回収できる伏線。

### 描かないほうがよい要素
- [ ] 歴史・制度・設定の長い講義。
- [ ] 登場人物の大量投入（目安：新規2〜3人まで）。
- [ ] 解決編に近い核心情報（犯人・真相・黒幕の断定）。

### 文字量／比率の目安
- 全体の**5〜10%以内**（長編でも短く）。
- 1シーン完結を基本にし、場面転換は多くても1回まで。

---

## 2. 第一章で描く範囲（下限〜到達点）

### 第一章の必達ゴール
第一章終了時点で、読者が次を読みたくなるために、最低限以下を満たす。

- [ ] 主人公が「誰で」「何に困っているか」が分かる。
- [ ] 主人公の当面の目的（短期目標）が提示される。
- [ ] 対立要素（人物・社会・環境・内面）の少なくとも1つが発火する。
- [ ] 「この話はどの方向へ進むか」の期待線が見える。

### 第一章で描くべき内容
- [ ] 主人公の日常と、その日常を崩す変化点（事件／出会い／命令）。
- [ ] 主人公が最初の意思決定を行う場面。
- [ ] 主要人物1〜2名との関係性の初期配置。
- [ ] 作品トーン（シリアス／コミカル／ダーク等）の確定。

### 第一章では描きすぎない内容
- [ ] 最終対立の全貌説明。
- [ ] 主要キャラ全員の過去回想。
- [ ] サブプロットの同時多発開始。

### 文字量／比率の目安
- 全体の**15〜25%程度**。
- 「導入→変化→最初の選択→小さな結果」までを1セットで完了。

---

## 3. プロローグ→第一章の接続ルール

- [ ] プロローグの問いのうち最低1つを第一章内で再提示する。
- [ ] 接続時に時系列・視点・語り口の変更理由を明確にする。
- [ ] プロローグの雰囲気と第一章の空気差が大きい場合、橋渡し1段落を置く。
- [ ] 第一章冒頭3〜5段落で「今誰の話か」を固定する。

---

## 4. 章設計テンプレート（実務用）

### プロローグ（テンプレ）
1. 異常の提示（読者に問いを残す）
2. 小さな決着または不穏の拡大
3. 本編へのフック文で締める

### 第一章（テンプレ）
1. 主人公の日常提示
2. 変化点の発生
3. 目的の設定
4. 最初の障害
5. 主人公の選択
6. 章末フック（次章への未解決要素）

---

## 5. 完了判定チェック（執筆後）

### プロローグ完了判定
- [ ] 一文で要約できる。
- [ ] 読者に残る問いが1〜2個に絞られている。
- [ ] 説明よりも出来事が先に来ている。

### 第一章完了判定
- [ ] 主人公の目的・障害・選択が本文で確認できる。
- [ ] 読者が次章で見たいものを具体的に想像できる。
- [ ] 情報過多になっていない（固有名詞を必要最小限に抑制）。

---

## 6. 運用メモ

- 短編：プロローグは省略可。第一章にフックを統合する。
- 長編：プロローグは「世界の異常」、第一章は「主人公の異常」を担当させると整理しやすい。
- 連載：第一話内で第一章の必達ゴールを満たすことを優先する。
