## ダメ出しを防ぐためのチェックリスト

### 1) 企画・設計（書き始め前）
- [ ] 作品の「一行要約（誰が・何を求め・何と戦うか）」を作った。
- [ ] 想定読者とジャンル期待（お約束）を明文化した。
- [ ] 物語の主題（テーマ）を1〜2文で言える。
- [ ] 主人公の欲求・欠落・変化を整理した。
- [ ] 敵対者（障害）の目的と論理を定義した。
- [ ] 世界観ルールと例外条件を先に決めた。
- [ ] 主要キャラの口調・語彙・価値観の差を設計した。
- [ ] 終盤の着地点（感情的ゴール）を仮決定した。

### 2) 構成・プロット
- [ ] 各章に「目的・障害・結果」の3点がある。
- [ ] 物語の中盤に新情報または関係変化を配置した。
- [ ] 山場（最低3箇所）が明確になっている。
- [ ] 伏線は「提示→忘却→回収」の流れで整理した。
- [ ] 偶然で始まり、解決は主人公の選択で終わる構成にした。
- [ ] クライマックス前に新設定を増やしすぎていない。
- [ ] エンディングは主要問いに回答している。

### 3) キャラクター
- [ ] 主人公が自ら決断する場面が十分にある。
- [ ] 各主要キャラに「欲望」「恐れ」「秘密」がある。
- [ ] 脇役は役割が重複しすぎていない。
- [ ] 人間関係の変化が段階的に描かれている。
- [ ] 年齢・職業に見合った言動になっている。
- [ ] 悪役にも納得可能な動機を与えた。

### 4) シーン運用
- [ ] シーン冒頭で「誰が・どこで・何をしたいか」を示した。
- [ ] 緊迫シーンで説明を挟みすぎていない。
- [ ] 会話文だけ／地の文だけに偏っていない。
- [ ] 五感描写を必要箇所に入れた。
- [ ] アクション時の位置関係が追える。
- [ ] 章末に読者の興味を次へつなぐ要素がある。

### 5) 文体・表現
- [ ] 一文の長さを調整し、読みやすいリズムを作った。
- [ ] 口調がキャラごとに識別できる。
- [ ] 同語反復を削減した。
- [ ] 比喩が多すぎず、意味が明確。
- [ ] 視点（POV）のルールを統一した。
- [ ] 誤字脱字・誤用を校正ツールと目視で確認した。

### 6) 整合性・設定検証
- [ ] 時系列表を作成し矛盾を潰した。
- [ ] 地理・移動時間・距離感を確認した。
- [ ] 専門設定（科学・歴史・法律等）を最低限調査した。
- [ ] 重要ルールが途中で変わっていない。
- [ ] 伏線回収漏れがないか一覧で確認した。

### 7) 感情設計
- [ ] 主人公の感情変化に「きっかけ」がある。
- [ ] 重大事件の後に余波シーンを置いた。
- [ ] 感動シーンで説明しすぎず行動で見せた。
- [ ] 恋愛・友情・対立の進展速度が適切。
- [ ] 読後に残したい感情を最終章で強化した。

### 8) 読者配慮・公開前
- [ ] ステレオタイプや偏見表現を見直した。
- [ ] 暴力・性描写の必要性と分量を再検討した。
- [ ] ジャンル期待を満たしつつ独自性を出せている。
- [ ] タイトルと内容の一致を確認した。
- [ ] 第三者（β読者）から3点以上の改善提案を得た。
- [ ] 指摘反映後に最終通読してテンポを確認した。

---

必要ならこのチェックリストを、
- **短編向け（5,000〜20,000字）**
- **長編向け（8万字以上）**
- **Web連載向け（話単位）**
の3種類に分割した実用版に作り替えられます。
