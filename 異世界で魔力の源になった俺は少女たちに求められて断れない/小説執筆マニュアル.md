# 小説執筆マニュアル

> AI（Cursor Agent）を活用した小説執筆の作業手順・注意事項・過去の教訓をまとめたドキュメント。
> 同種の作業で同じミスを繰り返さないために、**作業開始前に必ず通読すること**。

---

## 1. 全体ワークフロー

```
設定整理 → プロット作成 → シーン分割 → 並列執筆 → 結合 → 校正 → Git管理
```

### 各フェーズの概要

| フェーズ | 内容 | 成果物 |
|---------|------|--------|
| 設定整理 | 世界観・キャラ・ルールの構造化 | `小説舞台設定案.md` |
| プロット作成 | シーン単位の詳細プロット | `○○_プロット.md` |
| シーン分割 | 並列執筆用にシーンをグループ化 | 分割計画（4グループ以下） |
| 並列執筆 | Taskツールで同時執筆 | `本文/シーンX.md` |
| 結合 | Pythonスクリプトで結合 | `本文/○○_完成版.md` |
| 校正 | 表記揺れ・整合性チェック | 修正済み完成版 |
| Git管理 | commit & push | リポジトリ更新 |

---

## 2. ディレクトリ構成

```
N000001/                                ← リポジトリルート
├── README.md
└── {作品名フォルダ}/                   ← 作品ルートフォルダ
    ├── 小説舞台設定案.md               ← 世界観・キャラ設定
    ├── プロローグ_プロット.md          ← プロットファイル
    ├── 第1章_プロット.md               ← 章ごとにプロット作成
    ├── 小説執筆マニュアル.md           ← 本ファイル
    └── 本文/                          ← 執筆済み本文
        ├── プロローグ.md               ← シーン単位の分割ファイル
        ├── シーン4_5.md
        ├── ...
        ├── プロローグ_完成版.md        ← 結合済み完成版
        └── 第1章_完成版.md
```

**命名規則**:
- プロット: `{章名}_プロット.md`
- シーン分割: `シーンX.md` または `シーンX_Y.md`（複数シーンまとめる場合）
- 完成版: `{章名}_完成版.md`

## 3. 設定整理フェーズ

### やること
1. ユーザーのアイデアを `小説舞台設定案.md` に構造化して記録
2. 未決定の設定項目を洗い出し、ユーザーに確認 or 提案して埋める
3. キャラクター設定は**外見・性格・役割**を必ずセットで定義

### 注意点
- 設定ファイルは**1ファイルに集約**する（分散させない）
- テーブル形式を積極的に使い、一覧性を高める
- 設定追加・変更時は既存の構造に合わせて追記する（構造を壊さない）

---

## 4. プロット作成フェーズ

### やること
1. 章ごとにプロットファイルを作成
2. シーン単位で以下を定義:
   - 場所・時間帯
   - 登場人物
   - 起こる出来事（箇条書き）
   - 描写のポイント（何を読者に伝えたいか）
   - 感情の流れ（キャラの心理変化）

### 注意点
- 行為シーンがある場合は**描写方針を事前に明文化**しておく（設定ファイル内に記載）
- シーン間の繋がり（時間経過・場所移動）を明示する

---

## 5. 並列執筆フェーズ

### シーン分割ルール

**⚠️ 最重要: Taskツールは最大4エージェント同時起動**

- シーンが5つ以上ある場合は、関連シーンをグループ化して**4グループ以下**にまとめる
- 各グループは**独立して執筆可能**な単位にする
- グループ間の接続部分（前のシーンの最後の状況）は指示に明記する

### Taskツールへの指示テンプレート

```
あなたは小説「{作品名}」の執筆担当です。
以下のシーンを執筆してください。

## 作品情報
- ジャンル: {ジャンル}
- 文体: {文体の特徴}
- 視点: {一人称/三人称}

## 設定（必要な部分のみ抜粋）
{キャラ設定・世界観の関連部分}

## 執筆するシーン
{プロットから該当シーンを転記}

## 前のシーンの状況
{直前シーンの最後の状況を簡潔に}

## 描写方針
{行為シーンがある場合の描写ポリシー}

## 出力
- ファイルパス: {出力先}
- Markdown形式で出力
- シーン区切りには `---` を使用
```

### 注意点
- 各Workerへの指示は**完全に自己完結**させる
- 設定ファイル全文を渡すのではなく、**そのシーンに必要な設定だけ抜粋**して渡す
- 文体の統一感を保つため、サンプル文章を1段落分含めると効果的

## 6. ファイル結合フェーズ

### ⚠️ 最重要教訓: テキスト結合にAIエージェント（Task）を使わない

**テキストファイルの結合は必ずPythonスクリプトで行う。**

Taskツールでテキスト結合を試みると:
- 処理が非常に遅い
- エラーで中断するリスクがある
- 単純作業にAIリソースを浪費する

### 結合用Pythonスクリプト（テンプレート）

```python
import re

files = [
    r'本文/シーン1.md',
    r'本文/シーン2.md',
    r'本文/シーン3.md',
    # ... 結合順に列挙
]

parts = []
for f in files:
    with open(f, 'r', encoding='utf-8') as fh:
        parts.append(fh.read().strip())

# 区切り文字（作品に合わせて調整）
separator = '\n\n＊＊＊\n\n'
result = separator.join(parts)

# 出力
out = r'本文/○○_完成版.md'
with open(out, 'w', encoding='utf-8') as fh:
    fh.write(result)

# 統計表示
lines = result.count('\n') + 1
chars = len(result.replace('\n', '').replace(' ', ''))
print(f'完了！ 総行数: {lines}行 / 文字数(空白除く): 約{chars}字')
```

### カスタマイズポイント
- `separator`: 作品のシーン区切りに合わせる（`＊＊＊`、`---`、空行のみ、等）
- 不要な見出し（`## シーンX：〜`）の除去が必要な場合は `re.sub()` で処理
- 結合後に文字数を表示して進捗確認

## 7. PowerShell環境での注意事項

### UTF-8設定（毎回必須）
```powershell
$OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

### Git コミットメッセージ
**⚠️ PowerShellではHEREDOC構文が使えない**

```powershell
# NG: PowerShellでエラーになる
git commit -m "$(cat <<'EOF'
複数行のメッセージ
EOF
)"

# OK: シンプルな1行メッセージ
git commit -m "プロローグ執筆完了"

# OK: 複数行が必要な場合はダブルクォート内で `n を使う
git commit -m "プロローグ執筆完了`n`n- シーン1〜10を執筆`n- 完成版ファイルを作成"
```

### 日本語ファイル名
- Pythonスクリプト内では `r'文字列'`（raw string）を使う
- パスにスペースが含まれる場合はダブルクォートで囲む

## 8. 品質チェックリスト

### 執筆完了時の確認項目

- [ ] キャラクター名の表記揺れがないか（カタカナ統一）
- [ ] 一人称/三人称の視点が混在していないか
- [ ] 設定ファイルの内容と本文に矛盾がないか
- [ ] シーン間の時間経過・場所移動が自然か
- [ ] 行為シーンの描写が描写方針に沿っているか
- [ ] 結合後のシーン区切りが正しいか
- [ ] 文字数が想定範囲内か

### 表記揺れチェック用コマンド
```powershell
# キャラ名の揺れを検索（例）
rg "リーナ|りーな|Lina|lina" 本文/
```

## 9. 過去の教訓まとめ

| 教訓 | 詳細 | 対策 |
|------|------|------|
| テキスト結合にTaskを使うな | 処理が遅く、エラーで中断する | Pythonスクリプトを使う |
| PowerShellのHEREDOC | `$(cat <<'EOF'...)` が動かない | 1行メッセージ or バッククォート改行 |
| 設定は1ファイルに集約 | 分散すると整合性が取れなくなる | `小説舞台設定案.md` に全て記載 |
| Taskへの指示は自己完結 | 他Workerの成果物に依存させない | 必要な設定を全て指示に含める |
| シーン分割は4以下 | Taskツールの同時起動上限 | 関連シーンをグループ化 |
| 描写方針は事前に明文化 | 後から方針変更すると手戻りが大きい | プロット作成時に確定させる |

## 10. 新しい章を書く際のクイックスタート

1. **設定確認**: `小説舞台設定案.md` を読み、新キャラ・新設定があれば追記
2. **プロット作成**: `{章名}_プロット.md` を作成、シーン単位で詳細化
3. **分割計画**: シーンを4グループ以下に分割、各グループの担当範囲を決定
4. **並列執筆**: Taskツールで同時起動（指示テンプレートを使用）
5. **結合**: Pythonスクリプトで結合（Taskツールは使わない！）
6. **品質チェック**: チェックリストで確認
7. **Git**: commit & push（1行メッセージで）
