# フィニッシュ（射精）仕様 v1.0

**対象**: 泡児島ノベルゲーム  
**目的**: 「口での奉仕」を含む各体位／プレイ状況ごとに、適切なフィニッシュ選択肢を出す設計を定義する。  
**位置付け**: `docs/シーンテンポ仕様.md` の R1-3「絶頂直前モード」と R4「特殊体位・特殊プレイ」を具体化する補助仕様。

---

## 用語

| 用語 | 定義 |
|------|------|
| フィニッシュ | プレイヤーが射精する行為。シーン終了のトリガー |
| 着弾先 | フィニッシュ時の射精位置（中／口／顔／胸／腹／背中／お尻／外） |
| 我慢 | フィニッシュせずに継続。`pleasure` を一部消費して `piston` 継続 |
| 専用ステージ | 通常の体位ピストン以外（口でしてもらう／胸で挟む等）。`stage: 'oral'` / `'paizuri'` 等 |
| 同意フラグ | 中出し・口内射精・飲精など、好感度・経験で解放される行為の許可 |

## F3. キャラ別の許容ルール

`character-intimacy-profiles.js` に `finishPreference` を追加して制御する。

```js
finishPreference: {
  inside: 'unlocked' | 'requiresConsent' | 'forbidden',   // 中出し許可
  oralSwallow: 'eager' | 'shy' | 'reluctant' | 'forbidden',
  face: 'ok' | 'shy' | 'forbidden',
  preferredFinish: 'chest' | 'inside' | 'belly' | 'face',  // そのキャラが最も嬉しい着弾先
  reactionTone: { inside: '...', face: '...', oral_swallow: '...' }
}
```

### F3-1. キャラ別マトリクス（提案）

| キャラ | inside | oral_swallow | face | preferredFinish | 備考 |
|--------|--------|--------------|------|-----------------|------|
| 海凪（みなぎ） | requiresConsent | shy | shy | inside | 初回拒否、好感度60+で同意 |
| ひなた | unlocked | eager | ok | inside | 経験者、何でも受け入れる |
| さくら | requiresConsent | reluctant | forbidden | belly | 控えめ、お腹がギリ |
| あおい | unlocked | ok | ok | inside | 大人、どれもOK |
| みゆ | requiresConsent | shy | shy | chest | おっとり、胸が好き |
| りん | requiresConsent | eager | ok | face | 勝負系、顔も平気 |
| ももか | requiresConsent | shy | forbidden | inside | 上品、顔は禁忌 |
| かえで | requiresConsent | reluctant | shy | belly | お世話系 |
| ゆき | forbidden | forbidden | forbidden | belly | 最年少、お腹のみ |
| なぎさ | unlocked | ok | ok | inside | 大人、慣れている |
| ここあ | requiresConsent | eager | ok | chest | 明るい、胸射好き |

`forbidden` の選択肢は **画面に出さない**（UIをすっきり保つ）。

### F3-2. 同意の解放フロー

- `consent.inside`: 初回の中出しを選択すると `routeProgress.consentInside = true` になり、以降は自動許可
- ただし `finishPreference.inside === 'forbidden'` のキャラ（例: ゆき）には選択肢自体出さない
- `requiresConsent` のキャラには初回のみ確認ビートを挿入:

```
[システム]
本当に中に出しますか？

[1. 出す]   [2. やっぱり外に出す]
```

「出す」を選んだら以降は確認なし。

## F4. 効果（数値）

### F4-1. 共通効果

| 項目 | 効果 |
|------|------|
| 着弾先選択 | `affection += action.affection` |
| `preferredFinish` 一致 | `affection +3` ボーナス |
| `forbidden` を強行 | 不可（選択肢が出ない） |
| 我慢する | `pleasure -10`、`endurance +1`。3回目以降は強制フィニッシュ |
| ヒロインエンディング条件加算 | `inside` で `routeProgress.creampie = true` |

### F4-2. アクション後のビート構成（R2準拠）

フィニッシュアクションは **5〜7ビート**:

```
[A. 起こす描写]      ← 「腰の動きが加速する。もう、止まらない――」
[B. 彼女の悲鳴的反応] ← 「あぁっ、出てる、いっぱい……っ！」
[C. 着弾の描写]      ← 着弾先別の地の文
[D. 彼女の余韻]       ← 「……あったかい……」
[E. システム表記]     ← （好感度+XX / ルート進行）
```

特殊な着弾先（中出し・飲精）は **F. ヒロインの感想** を追加して7ビート。

## F5. 口での奉仕ステージ（`stage: 'oral'`）の発生

### F5-1. エントリー

前戯フェーズの選択肢に **「特別なこと」→「口でしてもらう」** を追加（R5 階層化）。

```js
foreplay.specialPlays = [
  { id: 'oral_service', label: '口でしてもらう', stage: 'oral', requires: { affectionMin: 30, zone: 'lips:1.2+' } }
```

### F5-2. ステージ内の選択肢


`canClimax = true` になったら**選択肢をフィニッシュメニューに切替**（F2-4 のセット）。

## F6. UI/UX

### F6-1. 選択肢表示

- フィニッシュ選択肢は **3〜5+我慢** の構成（R1-3 準拠）
- 「我慢する」は常に**最下段**に固定
- `forbidden` のものは選択肢にしない（出さない方が没入）

### F6-2. compactChoices モード

`canClimax === true` から実際に射精するまで:

- `body.classList.add('compact-choices')` で選択肢ボタンを大きく
- HUDを半透明化（`immersiveMode`）
- 画面を少し暗転

### F6-3. ナレーション

- 着弾先確定時に **1ビート暗転** を挟む（演出強化）
- 中出しのときだけ **赤い枠オーバーレイ** 等の強い演出（任意）
