# UI操作感仕様 v1.0

**対象**: 泡児島ノベルゲーム（ブラウザ）  
**目的**: ノベルゲームとして最低限満たすべきUI操作感を明文化し、実装・テストの基準とする。  
**位置付け**: `docs/画面設計書.md` の補足。本書は **操作感・UX の必須条件** を定義する。

---

## 用語

| 用語 | 定義 |
|------|------|
| メッセージ | `#message-window` に表示される台詞・地の文1ビート |
| 操作待ち | メッセージ全文表示後、次の処理に進むためユーザー入力を待つ状態 |
| ストーリー再生 | プロローグ／エンディング／チュートリアル（`gameEngine.storyActive === true`） |
| 通常メッセージ | ストーリー再生中ではない `uiManager.showMessage` 呼び出し |
| 選択肢 | `#choices-container` に表示される `.choice-button` 群 |
| 進行操作 | クリック / Enter / Space のいずれか（メッセージ進行用） |

---

## A. メッセージ表示

| ID | 仕様 | 受け入れ基準 |
|----|------|-------------|
| **A1** | メッセージは1ビートずつ表示する。**連続するメッセージは必ずユーザーの進行操作で進める**（自動連結しない） | 通常メッセージ→次のメッセージへ進むには進行操作が必要 |
| **A2** | タイピング中の進行操作 → **全文を即時表示**（次へは進まない） | タイピング中クリック1回で全文表示、もう1回で次へ |
| **A3** | 全文表示後の進行操作 → **次のステップへ進む**（操作待ちを解放） | クリック1回で次の `showMessage` / `showChoices` が動く |
| **A4** | 操作待ち中はメッセージ末尾に **「クリック / Enter で進む」** の進行ヒントを表示 | `#message-hint` テキストが更新される |
| **A5** | ストーリー再生中も A1〜A4 と同じ進行ルール（`waitingForStoryClick` → 進行操作で次ビート） | プロローグ各ビートで操作待ち |
| **A6** | メッセージ送信は冪等。重複クリックは1回分のみ消費 | 連打でビートを飛ばさない |

---

## B. 選択肢表示

| ID | 仕様 | 受け入れ基準 |
|----|------|-------------|
| **B1** | 選択肢は **画面の縦横中央** に固定表示する | リサイズしても常に中央 |
| **B2** | 選択肢が出る前のメッセージは **必ずユーザーの進行操作で進める**（自動で選択肢を表示しない） | キャラ自己紹介→「選びますか？」→選択肢、の各段でクリックが必要 |
| **B3** | 選択肢表示中は **オート/スキップを停止**する（誤クリックで進まない） | 選択肢が出ている間はタイマー走らない |
| **B4** | 選択肢クリック後はボタンを **即時 disabled** にし、ハンドラは1度しか動かない | 連打しても1回しか進まない |
| **B5** | 選択肢ボタンの隙間（コンテナ背景）のクリックは **無反応** | `pointer-events: none` on container |
| **B6** | 選択肢ハンドラで例外発生時は、進行不能にならず **直近の安全画面に復帰**（マップ or ハブ） | エラー時に `recoverFromChoiceError` 等が動く |
| **B7** | 選択肢間の遷移は背景・立ち絵を破壊しない（ちらつき最小） | キャラ画像が消えてから再表示される、等の不要な再描画なし |

---

## C. オート/スキップ

| ID | 仕様 | 受け入れ基準 |
|----|------|-------------|
| **C1** | **オート**: 全文表示後、テキスト速度設定に応じた遅延（遅3000 / 普通2000 / 速1000ms）で自動進行 | 設定どおりの間隔で次へ |
| **C2** | **スキップ**: 30msで自動進行。タイピング中は即全文表示してから即進行 | スキップONで連続消化 |
| **C3** | 選択肢表示中は **オート/スキップを自動停止**（B3） | 選択肢で待機が止まる |
| **C4** | ユーザーが手動進行しても **オート/スキップ状態は維持**（誤って解除しない） | クリックしてもON継続 |
| **C5** | Esc・メニュー表示・モーダル開閉時は **オート/スキップを即停止** | 安全な状態に戻る |
| **C6** | 通常メッセージの操作待ちも、オート/スキップ対象に含める | ストーリー再生外でも有効 |

---

## D. キーボード操作

| キー | 動作 |
|------|------|
| **Enter / Space** | メッセージ進行（A2/A3）。選択肢表示中は無効 |
| **1〜9** | 選択肢の1〜9番目をクリックと同等に選択 |
| **Esc** | モーダル open → 閉じる。なければメインメニュー表示。オート/スキップ即停止（C5） |
| **Ctrl+S** | クイックセーブ |
| **Ctrl+L** | クイックロード |

---

## E. モーダル

| ID | 仕様 | 受け入れ基準 |
|----|------|-------------|
| **E1** | モーダル表示中は **背景UI操作不可**（クリック貫通しない） | モーダル背景を消化 |
| **E2** | モーダルは **×ボタン / Esc / 背景クリック** で閉じる（既定） | 3手段すべて動く |
| **E3** | モーダル open 中はメッセージ送り・選択肢操作などの裏処理が **進まない** | 裏でビートが進まない |
| **E4** | **名前入力モーダル** は Esc / 背景クリックで閉じない（**必須入力**）。確定ボタンか Enter のみ閉じる | プレイヤー名未確定で進めない |
| **E5** | モーダル close 後、フォーカスは元の文脈（タイトル / メインゲーム）に戻る | キーボード操作が継続できる |

---

## F. レイアウト不変条件

| ID | 仕様 |
|----|------|
| **F1** | メッセージウィンドウ `#message-window` は常に **画面下部固定** |
| **F2** | 選択肢 `#choices-container` は常に **画面中央**（縦横）、解像度・本数問わず |
| **F3** | 上部HUD（場所・時間・日数・体力・行動）は常に **上部固定** |
| **F4** | キャラ立ち絵は **メッセージウィンドウ・選択肢の背面**（重ねない） |
| **F5** | 好感度HUD・性行為HUDは右側固定。マップ表示と重ならない |

---

## G. エラー・進行不能耐性

| ID | 仕様 |
|----|------|
| **G1** | 選択肢ハンドラ例外時、メインゲームに復帰してマップを表示（B6） |
| **G2** | データ欠損（台詞配列空・キャラID不正など）→ 既定メッセージで継続。クラッシュさせない |
| **G3** | 進行操作の二重発火は1回として扱う（A6・B4） |
| **G4** | ロード失敗・セーブ失敗時は **ユーザーに通知**し、ゲーム状態は維持 |
| **G5** | 画面遷移中の操作は **キューせず無視**（誤動作回避） |

---

## H. 適用範囲外（本仕様では規定しない）

- 文章演出のアニメーション（フェード等）の詳細パラメータ
- BGM/SE の音量・タイミング
- セーブデータのバイナリ仕様

## I. 自動テスト要件

| カテゴリ | テスト |
|----------|--------|
| A1〜A3 | `showMessage` 後に `awaitingUserAdvance === true` になる／`releaseUserAdvance` で `false` になる |
| B3 | 選択肢表示中は `scheduleAutoAdvanceIfNeeded` が早期 return する |
| B4 | クリック後ボタン `disabled` を確認 |
| B6 | `selectChoice` 内 try/catch でリカバリ呼び出し |
| C5 | `closeModal` 時に `skipAdvanceTimer` クリア |
| E4 | 名前入力モーダルは `modalIds` から除外 |
| G3 | `advanceMessage` の連続呼び出しで `pendingResolve` が1回のみ走る |
